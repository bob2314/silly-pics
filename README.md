# silly-pics

A caricature app scaffold with:

- **Desktop web app**: Node.js + Express + React + Vite, deployable on Vercel.
- **Companion iOS app scaffold**: SwiftUI + camera capture + Core Image caricature effects.

## Desktop app (macOS via browser)

Location: `/tmp/workspace/bob2314/silly-pics/desktop`

### Features
- Uses built-in camera (`getUserMedia`) to capture a portrait-style face image.
- **Quick Filter**: instant offline caricature via canvas-based distortion and style tuning.
- **AI Caricature**: real artist-style caricature generated by OpenAI's `gpt-image-1` image-edit endpoint, with `input_fidelity: high` to keep the subject recognizable.
- Random style option (with Express API endpoint fallback to local randomizer).
- Saves a branded image (`Silly Pics / Caricature Studio`) as PNG.

### Run locally

```bash
cd desktop
npm install
npm run dev
```

`npm run dev` runs Vite **and** mounts the Express handler from `api/index.js` as middleware on the same port, so `fetch('/api/caricature')` Just Works locally — no Vercel CLI required. Vite picks up `desktop/.env.local` automatically and mirrors keys into `process.env` for the API handler.

If you'd rather run the project under the actual Vercel platform (matches production routing exactly):
```bash
npm install -g vercel       # one-time
npm run dev:vercel          # alias for `vercel dev`
```
The first run will prompt you to link the directory to a Vercel project.

### Environment variables
- `OPENAI_API_KEY` — required for the AI Caricature feature.
  - **Locally**: put it in `desktop/.env.local` (gitignored). See `desktop/.env.example` for the format. Restart `npm run dev` after editing it.
  - **Deployed**: set it in your Vercel project settings under Environment Variables → Production (and Preview if you want PR deployments to work).
  - Without it, `/api/caricature` returns a clear 500 and the UI surfaces the error.

### Tuning the AI caricature output
The prompt sent to OpenAI lives in `desktop/api/index.js` as `buildCaricaturePrompt`. It uses layered prompt constants so the behavior is easier to tune:

1. `SYSTEM_STYLE_PROMPT`: fixed artist role and overall output promise.
2. `IDENTITY_PRESERVATION_PROMPT`: preserve likeness while exaggerating in a flattering way.
3. `FEATURE_EXAGGERATION_PROMPT`: emphasize eyes, smile, hair shape, glasses, and facial structure.
4. Optional `traits`: API callers can pass an array of extracted traits (for example `["clear-framed glasses", "goatee", "smiling expression"]`) to preserve and selectively exaggerate. The current UI does not perform a separate feature-extraction pass yet.
5. `ART_MEDIUM_PROMPT`: live-event caricature look, bold linework, painterly shading, vibrant colors, soft textured background.
6. `STYLE_PROMPT_HINTS`: per-preset palette/mood cue that changes with the active style or "Random Style" pick.
7. `MOOD_AND_COMPOSITION_PROMPT`: playful, polished, friendly, waist-up, centered, high detail.
8. `NEGATIVE_PROMPT`: no text, captions, watermarks, signatures, or logos.

To experiment, edit the fixed prompt constants (affects every generation), `STYLE_PROMPT_HINTS` (affects only the corresponding style preset), or eventually add a lightweight vision step that sends `traits` to `/api/caricature`.

### Build and lint
```bash
npm run lint
npm run build
```

### Vercel notes
- `desktop/vercel.json` includes rewrites for SPA routes and `/api/*` to the Express handler in `desktop/api/index.js`.
- The image-edit call can take several seconds. If you see timeouts on the Hobby tier, upgrade or set a `functions[].maxDuration` override.
